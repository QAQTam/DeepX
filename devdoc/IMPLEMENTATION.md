## Phase 7.1 — 审计持久化 (P0, +80 lines)

### 新增: `crates/deepx-tools/src/audit.rs` (+50 lines)

```rust
use sha2::{Sha256, Digest};
use serde::{Serialize, Deserialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AuditEntry {
    pub ts: u64,
    pub user: String,
    pub tool: String,
    pub action: String,
    pub args_hash: String,       // SHA-256 of JSON args
    pub result: String,          // "ok" | "blocked" | "error"
    pub elapsed_ms: u64,
    pub files: Vec<String>,
}

const AUDIT_MAX_BYTES: u64 = 10 * 1024 * 1024;  // 10MB

pub fn append_audit(entry: &AuditEntry);
pub fn audit_path() -> std::path::PathBuf;
fn rotate_if_needed();  // >10MB → gzip + truncate
```

关键模式: 参考 `persistence.rs` 的 `OpenOptions::new().create(true).append(true)`, 文件位置用 `platform::data_dir()`.

### 修改: `crates/deepx-tools/src/bridge.rs` (line ~149, +15 lines)

`ToolExecResult` 构造后追加 `append_audit()` 调用。

### 修改: `crates/deepx-tools/src/lib.rs` (+1 line)

`pub mod audit;`

### 边界情况
- Windows/NTFS 上 append 是原子的（每行独立 JSON，崩溃不污染）
- 10MB 上限 ~50K 条记录，超出触发 gzip 压缩
- `args_hash` 用 SHA-256，只存指纹不存 body

### 修改 `ToolHandler`

`safety: fn(&ToolCallCtx) -> SafetyVerdict` → `risk: ToolRisk`

删除 `default_allow()` 函数。

### 重写 `safety.rs`

```rust
pub struct SafetyPolicy;

impl SafetyPolicy {
    pub fn evaluate(risk: ToolRisk, in_workspace: bool) -> SafetyVerdict {
        match (risk, in_workspace) {
            (ReadOnly, _) => Allow,
            (Write, true) => Allow,
            (Write, false) => RequireAuth { reason: "write outside workspace" },
            (Destructive, true) => RequireAuth { reason: "destructive operation" },
            (Destructive, false) => Block("destructive outside workspace"),
            (Administrative, _) => RequireAuth { reason: "administrative operation" },
        }
    }
}
```

删除 `is_danger_command` 和 `classify_execution` — 被 `ToolRisk::Destructive` 取代。
